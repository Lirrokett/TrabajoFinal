2. Origen y contextualización

Este proyecto nace de la necesidad de digitalizar y optimizar los procesos de venta y control de stock en nichos de mercado especializados. En el contexto actual de transformación digital, las empresas requieren sistemas que aseguren la integridad de la información y la jerarquización de permisos.

El desarrollo se enmarca en un entorno de arquitectura de software orientada al cliente, donde se han priorizado la escalabilidad y la seguridad. El sistema responde a un modelo de negocio que requiere una diferenciación clara entre el flujo transaccional del cliente y la gestión de recursos empresariales (back-end administrativo), garantizando una persistencia de datos coherente mediante una infraestructura de base de datos relacional.

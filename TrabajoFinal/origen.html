2. Origen y Contextualización

La génesis de este proyecto reside en la necesidad de escalar los procesos de negocio dentro de un entorno altamente complejo como es el coleccionismo de cartas (Magic, Pokémon, Yu-Gi-Oh!). La disparidad de criterios en la clasificación de rarezas y la alta frecuencia de transacciones exigían una base de datos relacional normalizada y eficiente.

Técnicamente, el sistema se fundamenta en el estándar de persistencia JPA (Java Persistence API), lo que garantiza una gestión de datos escalable y segura. El contexto operativo se sitúa en un modelo híbrido: la aplicación permite la logística interna (reposición y apertura de stock) mientras ofrece una fachada comercial transparente para el cliente final, vinculando cada transacción a un historial auditable y un perfil de usuario único.

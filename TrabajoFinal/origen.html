2. Origen y contextualización del proyecto

Al trabajar en un entorno con una cantidad masiva de datos y categorías (Magic, Pokémon, etc.), es vital una organización técnica que evite errores de stock. Por ello, se plantea una aplicación que automatiza las tareas operativas más pesadas[cite: 48, 55, 58].

El proyecto se contextualiza en una arquitectura cliente-servidor utilizando JPA (Java Persistence API) para la persistencia. Esto permite que el registro de cada compra quede vinculado de forma permanente a un usuario y una fecha específica, facilitando auditorías y seguimientos legales de las ventas[cite: 1, 12, 14]. Por otro lado, la aplicación gestiona la logística interna, permitiendo transformar activos (como abrir cajas para obtener sobres) manteniendo la coherencia en la base de datos relacional[cite: 92, 94, 95].

4. Objetivos específicos

Nuestro proyecto se basa en una base de datos relacional con una interfaz gráfica intuitiva desarrollada en JavaFX[cite: 1, 31, 48]. En la aplicación existen dos niveles de acceso diferenciados:

  • Usuario Cliente: Puede realizar un CRUD sobre su carrito, modificar sus datos de perfil y gestionar su historial de compras con validación de seguridad bancaria[cite: 1, 12, 86].
  • Usuario Administrador: Tiene acceso al Panel de Reposición (Restock) para gestionar el inventario, abrir cajas para generar sobres o ejecutar un sistema de "Gacha" para cartas individuales[cite: 26, 83, 92].

Los filtros y consultas se realizan mediante queries optimizadas para obtener listados fáciles de leer, permitiendo que tanto el administrador como el cliente final tengan siempre una visión clara del estado del stock[cite: 48, 58, 61, 74].
